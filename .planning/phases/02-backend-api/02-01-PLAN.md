Create `backend/` directory with empty `__init__.py`.

Create `backend/inference.py` with these functions:

1. `load_model_and_data(model_path, data_path)`:
   - Loads model via `keras.models.load_model(model_path, compile=False)` to suppress metrics warning.
   - Builds multi-output activation model using Functional API rebuild pattern: `keras.Input(shape=(28,28,1))` -> iterate `model.layers` applying each layer -> collect outputs dict keyed by `layer.name` -> `keras.Model(inputs=inp, outputs=list(outputs.values()))`.
   - Loads MNIST test data from `data_path` (gzip + pickle with `encoding='latin1'`). Test set is index 2 of the 3-tuple. Reshape x_test to `(-1, 28, 28, 1)` as float32. Cast y_test to int.
   - Returns a dict: `{"activation_model": ..., "layer_names": [...], "test_images": x_test, "test_labels": y_test}`.
   - Default paths: `model_path="model/mnist_cnn.h5"`, `data_path` should use a path relative to project root or accept parameter. Use `Path(__file__).resolve().parent.parent` to compute project root for default paths.

2. `activation_to_base64(activation_map: np.ndarray, size: int = 64) -> str`:
   - Takes a 2D numpy array (H, W) -- a single filter's activation.
   - Normalizes to 0-255 using min-max. If max-min < 1e-8 (dead filter), return all-zeros image.
   - Creates PIL Image in mode 'L' (grayscale), resizes to (size, size) with NEAREST interpolation.
   - Saves to BytesIO as PNG, returns base64-encoded string.

3. `run_inference(image: np.ndarray, activation_model, layer_names: list) -> dict`:
   - Takes a single image with shape (1, 28, 28, 1) -- batch dim included.
   - Runs `activation_model.predict(image, verbose=0)` -- single pass gets all layer outputs.
   - Extracts softmax from last output `results[-1][0]` (shape (10,)).
   - Builds confidence array: list of 10 dicts `{"digit": i, "probability": round(float(softmax[i]), 6)}` sorted by digit 0-9 (per locked decision).
   - Predicted digit: `int(np.argmax(softmax))`.
   - Builds activations list: for each layer name containing 'conv', extract that layer's output `results[layer_idx][0]` (shape H, W, filters), iterate filters, call `activation_to_base64()` on each. Each activation entry: `{"layer_name": name, "shape": [H, W, filters], "maps": [base64_str, ...]}`.
   - Returns dict: `{"prediction": predicted_digit, "confidence": confidence_array, "activations": activations_list}`.

4. `image_to_base64(image: np.ndarray, size: int = 112) -> str`:
   - Takes a 2D numpy array (28, 28) of the input MNIST image.
   - Normalizes to 0-255, creates PIL Image mode 'L', resizes to (size, size) with NEAREST.
   - Returns base64-encoded PNG string.

Use these imports only (all available in conda ml env): `keras`, `numpy`, `gzip`, `pickle`, `io`, `base64`, `pathlib.Path`, `PIL.Image`.

Do NOT import tensorflow directly -- use `import keras` (Keras 3 is standalone in TF 2.19).
Do NOT divide MNIST pixel values by 255 -- Nielsen data is already [0,1].
