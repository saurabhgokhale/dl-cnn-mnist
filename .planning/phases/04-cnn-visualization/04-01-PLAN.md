**Step 1: Generate viridis LUT**

Create `frontend/app/lib/viridis.js` exporting a `VIRIDIS_LUT` array with exactly 256 entries, each an `[R, G, B]` array with integer values 0-255. Generate the values by running this Python script in the conda `ml` environment (which has matplotlib):

```bash
conda run -n ml python -c "
import matplotlib.cm as cm
import json
lut = []
for i in range(256):
    r, g, b, _ = cm.viridis(i / 255.0)
    lut.append([round(r * 255), round(g * 255), round(b * 255)])
print('export const VIRIDIS_LUT = ' + json.dumps(lut) + ';')
"
```

Write the output directly to `frontend/app/lib/viridis.js`.

Verify: The file has exactly 256 entries, first entry is approximately `[68, 1, 84]` (dark purple), last entry is approximately `[253, 231, 37]` (bright yellow).

**Step 2: Create ActivationHeatmaps component**

Create `frontend/app/components/ActivationHeatmaps.js` that:

- Accepts props: `activations` (array from backend response, each item has `layer_name`, `shape`, `maps[]`)
- For each layer in `activations`, render a section with:
  - Label: derive display name from layer_name. For "conv2d_1" show "Conv Layer 1 -- 26x26, 8 filters" using the `shape` field (shape[0]xshape[1], shape[2] filters)
  - A 2x4 grid (CSS grid, `grid-cols-4 gap-2`) of canvas elements
- Each canvas uses a `ViridisCanvas` sub-component (can be in the same file) that:
  - Uses `useRef` for canvas and `useEffect` to process the image
  - Takes `base64` prop (single activation map string) and `size` prop (default 64)
  - On mount/update: creates `new Image()`, sets `img.src = "data:image/png;base64," + base64`
  - In `img.onload`: draws image to canvas at `size x size`, reads `imageData`, maps each pixel's R channel through `VIRIDIS_LUT`, writes colored data back with `putImageData`
  - Canvas element has `style={{ imageRendering: "pixelated" }}` and `className="rounded"`
  - Canvas `width` and `height` attributes set to `size`
- Wrap each layer section in the existing card pattern: `bg-white rounded-xl shadow-sm border border-gray-200 p-6`
- If `activations` is null/undefined/empty, render nothing (return null) -- loading/error states handled in Plan 03

Important: Create `new Image()` INSIDE the useEffect, not outside. Include `[base64, size]` in the useEffect dependency array to avoid stale closures. The component is a client component -- add "use client" directive at top.
