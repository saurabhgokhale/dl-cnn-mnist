<task type="auto">
  <name>Task 2: Assemble page with API integration and state management</name>
  <files>frontend/app/page.js</files>
  <action>
Replace `frontend/app/page.js` with the complete interactive page:

1. Add `"use client"` directive at the top
2. Import useState, useEffect, useCallback from React
3. Import all four components from `./components/`
4. Define API_BASE: `const API_BASE = process.env.NEXT_PUBLIC_API_URL || "http://localhost:8000"`

5. State variables:
   - `result` (null initially) — full API response object
   - `loading` (false) — whether a fetch is in progress
   - `error` (null) — error message string

6. `fetchDigit` function (wrapped in useCallback with no deps):
   - Accepts optional `digit` parameter (null for random)
   - Sets loading=true, error=null
   - Builds URL: if digit !== null, append `?digit=${digit}` to `${API_BASE}/api/random-predict`
   - Fetches with native fetch, checks res.ok, parses JSON
   - Sets result on success
   - On catch: set error to a user-friendly message like "Could not reach the backend. Make sure the server is running on port 8000."
   - Finally: set loading=false
   - IMPORTANT: Do NOT clear result before fetch — previous result stays visible during loading (per locked decision)

7. useEffect on mount: call `fetchDigit()` for auto-load (per locked decision)

8. Render layout — vertical scroll, centered, max-w-3xl (approx 800px) mx-auto, px-4 py-8:
   - Header component
   - Error banner (if error): red-tinted card with error message and a "Retry" button that calls fetchDigit()
   - ImageSection with props: result, loading, onFetchDigit={(d) => fetchDigit(d)}, onFetchRandom={() => fetchDigit()}
   - PredictionSection with props: result
   - ActivationsPlaceholder

Use `flex flex-col gap-6` for vertical spacing between sections. The max-width container should have whitespace on sides (mx-auto handles this).
